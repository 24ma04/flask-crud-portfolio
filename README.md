# Flask CRUD アプリ（ポートフォリオ）

## 概要
- 本アプリケーションは、Webアプリケーション開発の一連の流れ（設計〜実装）を理解することを目的として作成した、Flask製のCRUDアプリケーションです。
- 前職ではカスタマーエンジニアとして現場対応に従事し、
「手作業・入力ミス・属人化した運用」がトラブルにつながる場面を多く経験しました。
その経験から、入力検証・処理の流れが明確なWebアプリを自ら設計・実装してみたいと考え、本成果物を作成しました。
- 開発実務は未経験ですが、
フロントエンド〜バックエンド〜データベースまでの基本的な連携と処理の流れを理解することを重視しています。

## 作成した理由・背景
- 現場業務では「決まった手順を人手で繰り返す」作業が多く、ミスが発生しやすい
- 運用を意識せずに作られたシステムが、現場負担を増やしているケースを見てきました。
- そのため
「誰が使っても同じ結果になる」「入力ミスを防げる」仕組みを、
Webアプリとして形にすることを目標としました。

## 工夫した点
- Flaskを用いた基本的なCRUD処理の実装方法
- フロントエンドとバックエンドの役割分担
- ORM（SQLAlchemy）を利用したデータ操作
- 入力チェックや削除確認など、誤操作を防ぐための仕組み
- Webアプリ全体の処理の流れ（リクエスト〜レスポンス）
**現場での運用を想定し「ミスが起きにくい作り」**を意識しました。

## 機能
- ログイン / ログアウト
- カード一覧表示
- 新規カード追加
- カード編集
- カード削除（削除確認ダイアログ付き）


※ 必要に応じて app.py 内でユーザーを追加してください。

## 技術スタック
- Python 3.x：学習・開発用の標準言語
- Flask：軽量で学習コストが低く、CRUDやテンプレート連携を理解するのに最適
- SQLAlchemy：PythonコードでDB操作でき、SQLを直接書かずに扱える
- SQLite：小規模アプリで設定不要、学習用に最適
- HTML / CSS：画面表示の基本構造とスタイル付け
- JavaScript：フォーム制御や動的表示のため

## 実行方法（Windows）

1. リポジトリをコピーする
※ すでにフォルダがある場合はこの手順は不要です
```
git clone https://github.com/24ma04/flask-crud-portfolio.git
```
2. 作成されたフォルダに移動する
```
cd flask-crud-portfolio
```

3. 仮想環境の作成（推奨）
```
python -m venv venv
```
4. 仮想環境の有効化(成功すると、プロンプトの先頭に (venv) が表示されます)
```
venv\Scripts\activate
```
5. 依存ライブラリのインストール

# requirements.txt がある場合(推奨)
```
pip install -r requirements.txt
```
# なければ個別にインストール
```
pip install Flask Flask-Login Flask-SQLAlchemy Werkzeug
```
6. アプリ起動
```
python app.py
```
7. ブラウザでアクセス
```
http://127.0.0.1:5000/login
```
